## 🔍 Problema del Link Público

### Verificar si funciona:
1. Ejecuta `LANZAR_TODO_WEB.bat`
2. Espera a que aparezca la URL pública
3. Copia la URL en otro dispositivo/navegador

### Si no aparece URL:
```batch
type logs\cloudflared.log
```

Busca:
- ✅ `https://xxxxx.trycloudflare.com` → funciona
- ❌ Errores de conexión → problema de red/firewall
- ❌ Sin URL → cloudflared falló

### Solución alternativa: ngrok
```batch
winget install ngrok.ngrok

:: En una terminal:
python web_app.py --host 127.0.0.1 --port 7860

:: En OTRA terminal:
ngrok http 7860
```

## 🆘 Troubleshooting Rápido

### Problema: "Aún tarda 20+ segundos"

**1. Verificar GPU en uso:**
```batch
nvidia-smi
```
Deberías ver `ollama.exe` usando VRAM. Si no:
- Reinstala drivers NVIDIA
- Verifica CUDA en `ollama serve`

**2. Limpiar modelos y recargar:**
```batch
ollama ps
ollama stop <nombre-modelo>
ollama run qwen2.5-coder:14b "test"
```

**3. Verificar RAM disponible:**
```batch
taskmgr
```
Si tienes <8 GB libres, cierra otras aplicaciones.

---

### Problema: "Error al llamar a Ollama"

**1. Verificar que Ollama está corriendo:**
```batch
ollama ps
```

Si no responde:
```batch
ollama serve
```

**2. Verificar modelos instalados:**
```batch
ollama list
```

Si falta alguno:
```batch
ollama pull qwen2.5-coder:14b
ollama pull qwen2.5-coder:7b
```

---

### Problema: "El chat carga pero no responde"

**1. Ver logs de la web:**
```batch
type logs\web_app.log
```

**2. Probar Ollama directamente:**
```bash
python orchestrator.py chat
```

Si funciona aquí pero no en la web → problema de comunicación web-Ollama.

Si no funciona ni aquí → problema de Ollama o modelos.

## 💡 Explicación Rápida

### ¿Por qué era tan lento?

1. **Búsqueda web automática**: Cada mensaje hacía 6 peticiones HTTP + descargaba 3 páginas (+10-15s)
2. **Pipeline doble**: Programador genera borrador COMPLETO → Arquitecto revisa TODO (+30-40s)
3. **GPU offload**: Capas GPU bajas = mucho trabajo en RAM = 10x más lento
4. **Modelos se descargan**: Sin `keep_alive`, cada respuesta recarga modelo (+15s)

### ¿Por qué las correcciones funcionan?

1. **Sin búsqueda web**: -10 segundos ⚡
2. **Modo rápido**: -25 segundos (una sola pasada) ⚡⚡
3. **Más GPU layers**: -70% tiempo (menos offload) ⚡⚡⚡
4. **Keep alive**: -15s en respuestas 2+

**Total**: De 60 segundos → 5 segundos = **12x más rápido** 🚀
